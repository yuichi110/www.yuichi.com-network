概要

L3転送

同じネットワーク内の通信はMACアドレスを使うL2転送で行われます。 一方、異なるネットワークに対する通信はL3転送と呼ばれるIPアドレスベースの転送処理がされます。

ホストとホストを直結してもL2転送がされますが、それだと2台以上接続できません。 そのため、L2転送を中継する機器であるL2スイッチに各ホストを接続します。

デフォルトゲートウェイ

通信の起点となる送信元ホストは宛先ホストのIPを確認して「自分と同じネットワークにいるかどうか」を判断します。 これには自分が持つIPアドレスとサブネットマスクが関わってきます。 たとえば「10.0.0.101/24」のIPを持つホストは「10.0.0.0/24」のネットワークに属しているため、 宛先IPが「10.0.0.102」であれば同一ネットワークだと判断し、「10.0.1.102」であれば別ネットワークだと判断します。

同じIPアドレスであっても、この判定はサブネットに依存します。 たとえばサブネットが異なる「10.0.0.101/16」というホストIPを持っていれば、 「10.0.0.102」だけでなく「10.0.1.102」も同一ネットワークになります。

同一ネットワークのホストに対する通信はそのホストに対してL2転送を直接行います。 一方、異なるネットワークに属するホストに対しては「 デフォルトゲートウェイ 」と呼ばれる機器に、 他のネットワークに転送してもらうかたちをとります。 このデフォルトゲートウェイには一般的にルーターや後述するL3スイッチが使われています。

ブロードバンドルーターに接続されたPCはDHCPでIPアドレスとサブネットマスクをもらうのと同時にデフォルトゲートウェイの指定も受けます。 デフォルトゲートウェイのアドレスにはブロードバンドルーターのIPが指定されています。 もしデフォルトゲートウェイが設定されていないと、PCは自分が加わっているネットワーク以外には一切通信ができません。

デフォルトゲートウェイにパケットを転送してもらうにはデフォルトゲートウェイに対してパケットを届ける必要があります。 この「ホストからデフォルトゲートウェイ」への通信は同じネットワーク内の通信ですので、L2転送が利用されます。 L2転送でパケットを受け取ったルーターなどが、そのパケットのIPアドレスを見てL3転送をします。

異なるネットワークへの通信にはL3転送が利用されますが、L3転送の裏側にはL2転送も存在していることを忘れないでください。 デフォルトゲートウェイにL2転送するためには、デフォルトゲートウェイのMACアドレスを解決する必要があります。 これはデフォルトゲートウェイのIPからARPで得ています。

Ciscoの機器でのデフォルトルートの設定

デフォルトルートの設定は後述するルーティングの設定をすることで実現します。 設定前は異なるネットワークに対してパケットを送れないため、pingに失敗します。

図: 構成図 

PC1#ping 10.0.1.1
Type escape sequence to abort.
Sending 5, 100-byte ICMP Echos to 10.0.1.1, timeout is 2 seconds:
.!!!!
Success rate is 80 percent (4/5), round-trip min/avg/max = 4/4/6 ms

PC1#ping 10.0.2.102
Type escape sequence to abort.
Sending 5, 100-byte ICMP Echos to 10.0.2.102, timeout is 2 seconds:
.....
Success rate is 0 percent (0/5)

細かいコマンドの説明はルーティングの設定にて扱いますが、 左側のホストでデフォルトルートをルーターのインターフェースである「10.0.1.1」に設定するには以下のようにします 

PC1#conf t
PC1(config)#ip route 0.0.0.0 0.0.0.0 10.0.1.1
PC1(config)#end

この設定をすることでPC1は自身が属する「10.0.1.0/24」のネットワーク以外へのパケットを、 10.0.1.1に対して転送するようになります。 同様の設定を右側のネットワークのPC2にも加えます。 右側のネットワークにいるホストにとって、左側のネットワークは異なるネットワークであるため、 こちらもデフォルトゲートウェイの設定が必要だからです。 

PC2#conf t
PC2(config)#ip route 0.0.0.0 0.0.0.0 10.0.2.1
PC2(config)#end

このようにすることで異なるネットワーク間でも通信が行えるようになります。 

PC1#ping 10.0.2.102
Type escape sequence to abort.
Sending 5, 100-byte ICMP Echos to 10.0.2.102, timeout is 2 seconds:
!!!!!
Success rate is 100 percent (5/5), round-trip min/avg/max = 5/6/7 ms

ルーティングテーブル(接続しているネットワーク)

ルーターがどのパケットをどのように転送するかは「パケットの宛先IPアドレス」と「 ルーティングテーブル 」で決まります。

ルーターはどのインターフェースからどのように転送するかという情報を持っています。 これはちょうどL2転送におけるMACアドレステーブルと似ていて、ルーティングテーブルと呼ばれています。 シスコのルーターですと、「 show ip route 」コマンドでそれを確認できます。 

R1#show ip route
Codes: L - local, C - connected, S - static, R - RIP, M - mobile, B - BGP
       D - EIGRP, EX - EIGRP external, O - OSPF, IA - OSPF inter area
       N1 - OSPF NSSA external type 1, N2 - OSPF NSSA external type 2
       E1 - OSPF external type 1, E2 - OSPF external type 2
       i - IS-IS, su - IS-IS summary, L1 - IS-IS level-1, L2 - IS-IS level-2
       ia - IS-IS inter area, * - candidate default, U - per-user static route
       o - ODR, P - periodic downloaded static route, H - NHRP, l - LISP
       a - application route
       + - replicated route, % - next hop override, p - overrides from PfR

Gateway of last resort is not set

      10.0.0.0/8 is variably subnetted, 4 subnets, 2 masks
C        10.0.1.0/24 is directly connected, GigabitEthernet0/1
L        10.0.1.1/32 is directly connected, GigabitEthernet0/1
C        10.0.2.0/24 is directly connected, GigabitEthernet0/2
L        10.0.2.1/32 is directly connected, GigabitEthernet0/2

重要なのが出力の最後にある4行です。 ここでは「10.0.1.0/24はG0/1インタフェースに直接接続されている」ことと、 「10.0.2.0/24」はG0/2インターフェースに直接接続されている」ことがわかります。 また、自身の持つIPも「10.0.1.1/32はG0/1に直接接続されている」こともわかります。 「10.0.2.1/32」も同様にG0/2に直接接続されています。

一番左にある「C」や「L」は「show ip route」コマンドの前半に意味が書いてあり、 CはConnectedの略で直接接続ネットワークの際に使われて、 LはLocal(リンクローカルとも呼ばれる)の略で自分自身のIPの際に使われます。

これらのリンクローカルや直接接続の設定はルーターのインターフェースにサブネット付きでIPを与えれば、 自動で設定がされます。

ルーティングテーブルの参照

ルーターはパケットをL2転送で受け取った際に、 このルーティングテーブルの内容を参照することでどこから出せばいいかを判断します。

たとえばPC1からPC2宛の通信をG0/1インターフェースで受け取れば、 宛先IPアドレスが「10.0.2.102」であることから、 「10.0.2.0/24」が接続されるインターフェースから出せばよいことがわかります。

ルーターから宛先ホストへのパケット転送も同じネットワーク(10.0.2.0/24)内であるため、 MACアドレスを使ったL2転送がおこなわれます。 IPアドレスのヘッダに大きな変化はありませんが、 イーサネットのヘッダは「10.0.1.1」で受け取ったものと「10.0.2.1」から出すものとでは全く異なっています。

ルーターからのパケットを受け取ったPC2がPC1にパケットを送り返す際も全く同じ流れとなります。 PC2からパケットを受け取ったルーターは宛先IPアドレスがPC1であることを知り、 ルーティングテーブルを参照して「10.0.1.0/24」がG0/1インターフェースに接続していることに気付いて、 G0/1からPC1にパケットを転送します。 帰りのパケットもイーサーネットヘッダーの付け替えが行われます。

ルーティングテーブルに記載されない宛先ネットワークのホストへのパケットが届いた場合は、 そのパケットは破棄されます。 たとえば「10.0.3.103」宛のパケットをこのルーターが受け取れば、そのパケットは破棄されますので、 どのインターフェースからも出されることはありません。

ルーティングテーブルにエントリがない場合はping(ICMP Echo Request)に対して、 アンリーチャブルという返答をします。 

PC1>ping 10.0.2.102
Type escape sequence to abort.
Sending 5, 100-byte ICMP Echos to 10.0.2.102, timeout is 2 seconds:
U.U.U
Success rate is 0 percent (0/5)

ルーティングテーブル(接続していないネットワーク)

ルーターは直接接続されないネットワークあてのパケットを転送する必要もあります。 以下の図にあるように左側のルーター1は右側のネットワーク10.0.2.0/24に接続していません。

図を見れば右側のルーター2に転送をすればパケットを届けられることは一目瞭然ですが、 ルーターはルーティングテーブルのエントリという形で「10.0.2.0/24へはルーター2を経由する」ということを知っている必要があります。

自分が接続しないネットワークへの転送方法の定義ですが、 自分で定義する「 スタティックルート 」と呼ばれる方法と、 ルーター間で会話して自動で定義される「 ダイナミックルート 」と呼ばれる手法の2つがあります。 これらのどちらも設定されていないと、ルーティングテーブルには直接接続のネットワークしか記載されません。 

R1# show ip route
省略

      10.0.0.0/8 is variably subnetted, 7 subnets, 2 masks
C        10.0.1.0/24 is directly connected, GigabitEthernet0/1
L        10.0.1.1/32 is directly connected, GigabitEthernet0/1
C        10.0.99.0/24 is directly connected, GigabitEthernet0/2
L        10.0.99.1/32 is directly connected, GigabitEthernet0/2

ダイナミックルートについては別ページにて扱うため、 ここではスタティックルートに絞って説明をします。

スタティックルートは管理者がルーターに対して「このネットワーク宛の通信はここに送る」と定義をすることで、 直接接続されないネットワークへの転送を実現します。 上記の図ではルーター1は「10.0.2.0/24」にパケットを送るには、 ルーター1に隣接するルーター2のインターフェース「10.0.99.2」に対してパケットを送ればよいということです。

スタティックルートを設定するには「ip route <ネットワークアドレス> <サブネットマスク> 中継するIP」とします。 

R1(config)#ip route 10.0.2.0 255.255.255.0 10.0.99.2

このようにすることで、ルーティングテーブルが以下のように変化します。 

R1#show ip route
Codes: L - local, C - connected, S - static, R - RIP, M - mobile, B - BGP
       D - EIGRP, EX - EIGRP external, O - OSPF, IA - OSPF inter area
       N1 - OSPF NSSA external type 1, N2 - OSPF NSSA external type 2
       E1 - OSPF external type 1, E2 - OSPF external type 2
       i - IS-IS, su - IS-IS summary, L1 - IS-IS level-1, L2 - IS-IS level-2
       ia - IS-IS inter area, * - candidate default, U - per-user static route
       o - ODR, P - periodic downloaded static route, H - NHRP, l - LISP
       a - application route
       + - replicated route, % - next hop override, p - overrides from PfR

      10.0.0.0/8 is variably subnetted, 7 subnets, 2 masks
C        10.0.1.0/24 is directly connected, GigabitEthernet0/1
L        10.0.1.1/32 is directly connected, GigabitEthernet0/1
S        10.0.2.0/24 [1/0] via 10.0.99.2
C        10.0.99.0/24 is directly connected, GigabitEthernet0/2
L        10.0.99.1/32 is directly connected, GigabitEthernet0/2

エントリの3つめに「10.0.2.0/24」は「10.0.99.2」を経由(via)というものが増えています。 これがスタティックルートと呼ばれる設定手法で、スタティックルートで設定されたという印に一番左に「S」とあります。 出力上部のコードを見ると「S」は「static」とあります。

このようにすることでルーター1は10.0.2.0/24宛のパケットをルーター2に転送するようになります。 ルーター2にも同様に10.0.1.0/24宛のパケットは10.0.99.1を経由するとスタティックルートに記載します。 

R2(config)#ip route 10.0.1.0 255.255.255.0 10.0.99.1

こうすることでR2のルーティングテーブルは以下のようなものとなります。 

R2#show ip route
省略
      10.0.0.0/8 is variably subnetted, 7 subnets, 2 masks
S        10.0.1.0/24 [1/0] via 10.0.99.1
C        10.0.2.0/24 is directly connected, GigabitEthernet0/1
L        10.0.2.1/32 is directly connected, GigabitEthernet0/1
C        10.0.99.0/24 is directly connected, GigabitEthernet0/2
L        10.0.99.2/32 is directly connected, GigabitEthernet0/2

これで10.0.1.101のPC1と10.0.2.102のPC2の間で通信ができるようになりました。 

PC1>ping 10.0.2.102
Type escape sequence to abort.
Sending 5, 100-byte ICMP Echos to 10.0.2.102, timeout is 2 seconds:
!!!!!
Success rate is 100 percent (5/5), round-trip min/avg/max = 14/16/20 ms

デフォルトルート

先ほど説明したようにルーティングテーブルにエントリがない宛先へのパケットはルーターで破棄されてしまいます。 「 デフォルトルート 」はデフォルトゲートウェイのルーター版であり、 特別に指定されていない宛先は全てこのアドレスに転送をするというものです。

スタティックルートの書き方で「10.0.1.0 255.255.255.0」とすると「10.0.1.0/24」のことを表します。 そして「10.0.0.0 255.255.0.0」とすれば「10.0.0.0/16」のことを表します。 「10.0.0.0/16」は「10.0.1.0/24」のことも含んでいますので、より大きな範囲指定をしています。 このルーティングテーブルで書かれるエントリのネットワークは必ず存在しているという必要はなく、 あくまでも転送ルールにすぎません。

この大きな範囲のネットワークのかきかたをつきつめると「0.0.0.0 0.0.0.0」となります。 これは「全てのアドレス」を意味しているため、どのIPアドレスに送るにしてもこのエントリに該当するようになります。

実はホストとして使っているルーターに設定していたデフォルトゲートウェイの設定は、 デフォルトルートの設定となります。 

PC1#conf t
PC1(config)#ip route 0.0.0.0 0.0.0.0 10.0.1.1
PC1(config)#end

ルーティングテーブル 

10.0.0.0/8 is variably subnetted, 4 subnets, 2 masks
C        10.0.1.0/24 is directly connected, GigabitEthernet0/1
L        10.0.1.1/32 is directly connected, GigabitEthernet0/1
C        10.0.2.0/24 is directly connected, GigabitEthernet0/2
L        10.0.2.1/32 is directly connected, GigabitEthernet0/2

S 0.0.0.0/0 via 10.

詳しくは別ページで扱いますが、ルーティングテーブルのエントリは重複が許されていて、 重複している場合は「 最も優先度がエントリに従って転送をする 」というルールがあります。 上記で言えば「0.0.0.0/0」は全てのネットワークのことなので、 「10.0.1.0/24」や「10.0.2.0/24」も含まれてしまうため重複しているといえます。

ただ、「10.0.2.0/24」のエントリは「0.0.0.0/0」より優先度が高いため、 10.0.2.0/24宛の通信は「10.0.2.0/24」のエントリに従って転送されます。 一方、10.0.3.0/24宛の通信は条件に合致するのは「0.0.0.0/0」だけなので、このエントリに従って転送されます。

家庭用のブロードバンドルーターは実はこのデフォルトルートだけを持っています。 本来のルーターは様々なネットワークに接続してルーティングテーブルのエントリに従って複雑な転送ルールに従って転送をしますが、 ブロードバンドルーターは自分が直接接続する家庭内のネットワーク及びサービスプロバイダー側のネットワークのアドレス以外は、 全てサービスプロバイダーのルーターに転送をします。

一般的な家庭にはインターネットへの接続口がサービスプロバイダーしかなく、家のネットワークも1つしかありません。 そのため、全てをサービスプロバイダーのルーターに任せるという動きで十分なのです。
